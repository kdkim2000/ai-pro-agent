## 3. 발생 이슈 및 해결

### Issue 1: `factory.py` 미생성 상태에서 작업 시작

| 항목 | 내용 |
|------|------|
| **증상** | T1-2 수행 이력 없음. 실 커넥터·Mock은 구현되어 있으나 `ConnectorFactory`와 테스트가 누락된 상태 |
| **원인** | 이전 세션에서 커넥터 구현까지만 진행하고 Factory, 테스트 생성 전 세션 종료 |
| **해결** | `src/connectors/` glob → 누락 파일 식별 → `factory.py` 및 `tests/connectors/` 신규 생성 |
| **교훈** | 세션 시작 시 **현황 파악(파일 목록 확인)을 먼저 수행**하면 중복 작업 없이 이어받기 가능 |

### Issue 2: `git log` 에러 (커밋 없는 저장소)

| 항목 | 내용 |
|------|------|
| **증상** | `git log --oneline` 실행 시 `fatal: your current branch 'main' does not have any commits yet` |
| **원인** | `git init`만 되어 있고 커밋이 없는 상태 |
| **해결** | 에러 무시하고 `git add` → `git commit` 순서로 진행 |
| **교훈** | `git status`와 `git log`를 함께 실행 시 후자의 에러가 전자 확인을 방해하지 않도록 분리 실행 |

## 5. 학습 포인트 (VIBE 코딩 패턴)

### 5.1 세션 이어받기 패턴

- 새 세션 시작 시 `glob src/connectors/**/*.py` → 파일 목록으로 현황 파악
- 가이드 완료 기준(`§1.4 Definition of Done`)과 실제 파일 목록을 대조해 누락 식별
- **CLAUDE.md의 진행 현황 섹션**이 이어받기의 핵심 앵커

### 5.2 Factory 패턴 + 환경변수 전환

- `USE_MOCK_CONNECTORS=true` 한 줄로 실/Mock 전환
- T3 Skill 개발 시 실 시스템 없이 개발 → 검증 단계에서 `false`로 전환
- 이 패턴 덕분에 테스트가 CI/CD 파이프라인에서 네트워크 없이 실행 가능

### 5.3 테스트 설계 원칙

- **Mock 기반 테스트는 비즈니스 로직 검증**: 반환 타입, 필드 존재, 데이터 의미 확인
- 실 연동 테스트(`test_github_real.py` 등)는 별도 파일로 분리 → `skipif` 데코레이터로 Mock 모드 시 자동 skip
- `os.environ["USE_MOCK_CONNECTORS"] = "true"` 를 테스트 파일 최상단에서 모듈 임포트 전에 설정 (순서 중요)

### 5.4 git 초기 push 체크리스트

1. `git status --short` 로 `.env` 미포함 확인 (A 목록에 없어야 함)
2. 특정 파일 명시 스테이징 (`git add file1 dir/` 방식) — `git add .` 보다 안전
3. push 전 `git check-ignore -v .env` 로 gitignore 규칙 적용 검증
