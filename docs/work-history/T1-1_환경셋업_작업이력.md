## 3. 발생 이슈 및 해결

### Issue 1: T1-1_GUIDE.md 빈 파일

| 항목 | 내용 |
|------|------|
| **증상** | 최초 `T1-1_GUIDE.md` 조회 시 빈 파일 (0 bytes) |
| **원인** | 사용자가 아직 가이드 내용을 작성하지 않은 상태 |
| **해결** | 사용자에게 확인 요청 → 사용자가 999줄 가이드 업데이트 후 재진행 |
| **교훈** | 참조 문서가 비어있을 수 있으므로 반드시 내용 확인 후 작업 시작 |

### Issue 2: requirements.txt 인코딩 오류

| 항목 | 내용 |
|------|------|
| **증상** | `pip install -r requirements.txt` 실행 시 `UnicodeDecodeError: 'cp949' codec can't decode byte 0xe2` |
| **원인** | requirements.txt에 한글 주석(`# Agent 오케스트레이션` 등) 포함, pip가 cp949로 디코딩 시도 |
| **해결** | 한글 주석을 영문으로 변경 (`# Agent Orchestration` 등) |
| **교훈** | Windows 환경에서 pip는 시스템 기본 인코딩(cp949)으로 파일을 읽음. **requirements.txt에 한글 주석 사용 금지** |

### Issue 3: pip 구버전

| 항목 | 내용 |
|------|------|
| **증상** | Python 3.11.0 기본 pip 22.3 → 최신 패키지 다운로드 시 호환성 경고 |
| **해결** | `python -m pip install --upgrade pip` → 26.1.2로 업그레이드 후 재설치 |
| **교훈** | 가상환경 생성 직후 `pip upgrade`를 먼저 수행하는 것이 좋음 |

## 7. 학습 포인트 (VIBE 코딩 패턴)

### 7.1 가이드 문서 기반 코드 생성
- T1-1_GUIDE.md에 구현 코드가 포함되어 있어, 가이드의 코드를 **그대로 반영**하는 방식으로 진행
- 가이드 문서가 상세할수록 VIBE 코딩의 정확도와 속도가 향상됨
- 가이드의 섹션 번호(§4.1, §5.1 등)를 코드 docstring에 기록하여 **추적성 확보**

### 7.2 순서 의존성 관리
- `__init__.py` → 핵심 코드 → 테스트 코드 순서로 생성
- 패키지 초기화가 선행되어야 import가 정상 동작
- `pytest.ini`의 `pythonpath = .` 설정이 핵심 (프로젝트 루트에서 src 모듈 import 가능)

### 7.3 인코딩 이슈 대응
- Windows 환경에서 pip, Python 스크립트의 기본 인코딩이 cp949
- **requirements.txt, 설정 파일에는 한글 주석을 피할 것**
- 소스 코드(.py)는 UTF-8로 작성되므로 한글 docstring 사용 가능

### 7.4 점진적 검증
- 전체를 한 번에 만들고 마지막에 검증하지 않음
- **사내 API 없이 검증 가능한 테스트(test_scaffold.py)를 먼저 실행**하여 구조적 문제를 조기 발견
- 사내 API 연동 테스트(test_smoke.py)는 SCP/VDI 환경으로 전환 후 실행

### 7.5 Task 추적
- `task.md` 체크리스트로 진행 상황을 실시간 추적
- 각 단계 완료 시 `[ ]` → `[x]`로 업데이트
- 진행 중 항목은 `[/]`로 표시하여 현재 작업 위치 파악 가능
